# LocalPlayer V1 开发计划

## 架构决定

- 使用 SwiftUI 构建界面。
- 使用 SwiftData 保存歌曲索引；MP3 文件本体保存在 App 沙盒。
- 使用 SHA-256 判断文件是否重复；只有哈希一致才拒绝导入。
- 使用 AVFoundation 播放本地 MP3。
- 使用 MediaPlayer 提供锁屏和控制中心控制。
- 使用 XcodeGen 从 `project.yml` 生成 Xcode 工程。
- 每完成一项任务：验证、Git 提交，再开始下一项。

## 阶段一：工程能构建

### 任务 1：最小 iPhone App 与云端构建

- 内容：补全 XcodeGen 工程、创建空 SwiftUI App 入口、配置 Git 忽略规则和 GitHub Actions。
- 验收：macOS 环境能运行 `xcodegen generate`，并成功构建 iOS 模拟器 App。
- 验证：GitHub Actions 构建通过。
- 依赖：无。

## 阶段二：本地音乐库

### 任务 2：定义歌曲数据与本地索引

- 内容：定义 `Song` 数据模型，保存文件哈希、本地路径、歌名、歌手、时长和封面信息。
- 验收：歌曲记录可保存、读取；元数据缺失时使用规格中的默认值。
- 验证：单元测试覆盖默认歌名、未知歌手和排序。
- 依赖：任务 1。

### 任务 3：导入 MP3 并防止重复

- 内容：调用系统文件选择器，复制 MP3 到 App 沙盒，流式计算 SHA-256，提取元数据并创建歌曲记录。
- 验收：可导入多个 MP3；重复文件不产生重复歌曲；原文件不被修改。
- 验证：导入逻辑单元测试；真机手动导入。
- 依赖：任务 2。

### 任务 4：歌曲库界面与删除

- 内容：显示歌曲列表，支持默认排序、空状态、导入入口和删除 App 内副本。
- 验收：重启后歌曲仍显示；删除后文件与索引均消失，原始文件仍在“文件”App。
- 验证：模拟器 UI 测试与真机手动测试。
- 依赖：任务 3。

## 阶段三：播放体验

### 任务 5：本地播放与后台持续播放

- 内容：实现播放、暂停、进度、音量、上一首/下一首，并配置后台音频会话。
- 验收：选中歌曲可播放；锁屏后不中断。
- 验证：播放状态单元测试；真机锁屏测试。
- 依赖：任务 4。

### 任务 6：完整播放器界面

- 内容：实现当前歌曲信息、封面、进度条和播放控制。
- 验收：界面状态始终与实际播放状态一致。
- 验证：模拟器 UI 测试；真机手动播放测试。
- 依赖：任务 5。

### 任务 7：锁屏与控制中心

- 内容：发布当前歌曲信息，并处理播放、暂停、上一首、下一首远程命令。
- 验收：锁屏和控制中心能显示歌曲并控制播放。
- 验证：真机锁屏与控制中心测试。
- 依赖：任务 5。

## 阶段四：交付 IPA

### 任务 8：生成 IPA 构件

- 内容：GitHub Actions 生成未签名的 iPhone IPA，并作为构件保存。
- 验收：你能下载 IPA，使用 Sideloadly 安装到自己的 iPhone。
- 验证：实际安装、导入 MP3、锁屏播放完整走通。
- 依赖：任务 1 至 7。

## 检查点

- 阶段一结束：工程可被云端 macOS 构建。
- 阶段二结束：本地 MP3 可导入、持久化、显示和删除。
- 阶段三结束：播放器、后台播放、锁屏控制全部可用。
- 阶段四结束：IPA 可安装并通过完整验收。

## 范围纪律

- 不在 V1 加入播放列表、歌词、收藏、均衡器或其他格式。
- 不提交 MP3、IPA、证书、Apple ID 或签名文件。
- 每项任务只在上一项验证通过后开始。
