# ADR-001：使用歌曲上的扁平分类名称

## 状态

已接受，2026-07-13。

## 背景

V1.1 需要给本地歌曲分组、重命名分类和删除分类。项目是仅供个人使用的本地音乐播放器，分类没有层级、共享、权限或独立元数据需求。

## 决策

在 `Song` 上新增可选 `categoryName`。`nil` 表示“未分类”；分类列表从歌曲记录中汇总。分类改名与删除以批量更新匹配歌曲记录实现。

## 考虑过的替代方案

### 独立 `Category` SwiftData 模型

- 优点：可以保存空分类，关系更显式。
- 缺点：需要额外关系、迁移和删除规则；对单层、仅个人使用的分类来说没有足够收益。
- 结论：不采用。

### 直接修改 MP3 的标签

- 优点：分类和歌名能被其他播放器读取。
- 缺点：会修改用户的音乐文件，违背当前项目的本地文件保护边界。
- 结论：不采用。

## 后果

- 分类只能在至少一首歌曲使用时存在。
- 所有分类管理操作都必须规范化名称，并在 SwiftData 保存后才更新界面。
- 将来若需要空分类、封面或排序，可新增独立模型并另写迁移方案。
