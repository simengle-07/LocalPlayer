# LocalPlayer V1.1 实施计划

**状态：待实施计划批准**

## 架构决策

- 分类保存为 `Song.categoryName: String?`；`nil` 即“未分类”。不新增 `Category` SwiftData 模型，避免为单用户、单层分类引入关系迁移和空分类状态。
- 改名复用 `Song.title`，只影响 App 索引。
- iPhone 适配通过 `Info.plist` 的启动屏和 Targeted Device Family 修复，不通过硬编码 16 Pro 尺寸修补界面。

## 任务

### 任务 9：修复 iPhone 完整屏幕配置

- 内容：在 XcodeGen 配置中生成启动屏、将目标限制为 iPhone，并让 CI 断言归档 IPA 的设备族、启动屏和后台音频键。
- 验收：归档中的 `UIDeviceFamily` 为 `[1]`；`UILaunchScreen` 存在；`UIBackgroundModes` 含 `audio`。
- 验证：GitHub Actions 构建、测试、归档检查通过；真机 iPhone 16 Pro 显示完整窗口。
- 文件：`project.yml`、`.github/workflows/ios-build.yml`。

### 任务 10：增加歌曲分类与改名的数据能力

- 内容：为 `Song` 增加可选分类；提供纯逻辑辅助方法，用于规范化名称、计算分类、重命名和移动分类。
- 验收：空白歌名与空白分类不会保存；既有记录默认“未分类”；分类名称比较忽略大小写。
- 验证：先添加失败的单元测试，再实现并通过完整单元测试。
- 文件：`Models/Song.swift`、`LocalPlayerTests/SongTests.swift`。

### 任务 11：完成分类浏览与歌曲编辑界面

- 内容：在音乐库加入“全部 / 未分类 / 自定义分类”筛选；给歌曲行加入重命名、移动分类、改名分类和删除分类操作；队列跟随当前过滤列表。
- 验收：分类切换立即过滤；改名、移动和删除分类在重启后保持；正在播放栏在筛选后仍可打开播放器。
- 验证：CI 单元测试通过；iPhone 真机完成一轮“创建分类 → 移动歌曲 → 重命名 → 删除分类”。
- 文件：`App/LibraryView.swift`、新增的轻量 SwiftUI 编辑/筛选组件（最多三个）、`Services/AudioPlayerService.swift`（仅当需要同步队列）。

### 检查点：核心功能

- CI 通过；新 IPA 的 plist 键正确。
- iPhone 16 Pro 没有兼容尺寸黑边。
- 分类、改名与后台播放同时可用。

## 风险与处理

| 风险 | 处理 |
| --- | --- |
| SwiftData 旧记录没有分类 | 以可选字段迁移，`nil` 直接表示“未分类”。 |
| 分类筛选后切歌越界 | 播放队列始终使用当前过滤列表，并保留既有边界判断。 |
| 启动屏修复未进入 IPA | CI 直接读取归档内的 Info.plist，缺键即失败。 |

## 不在本次范围

- 多级文件夹、空分类、播放列表、批量编辑、iPad 适配、MP3 内部标签编辑。
