# LocalPlayer V1.3 规格：两级音乐分类

**状态：已实施，等待 GitHub Actions 与真机升级验证**
**日期：2026-07-13**

## 目标

将目前的平铺分类扩展为固定两级：一级分类下可有二级分类。每首歌曲只能属于一个位置：未分类、仅一级分类，或某个一级分类下的二级分类。

示例：

```text
日语音乐
├── 动画歌曲
└── 游戏原声

轻音乐
```

歌曲可属于“日语音乐 / 动画歌曲”，也可直接属于“轻音乐”；不支持第三层分类、多个分类或标签。

## 已确认的规则与假设

1. 分类层级固定为两级，不做无限层级树。
2. 一首歌曲最多有一个一级分类和一个二级分类；设置二级分类时必须同时有一级分类。
3. 继续使用现有 `Song.categoryName` 保存一级分类，新增可选 `Song.subcategoryName` 保存二级分类。旧歌曲的 `categoryName` 保留，自动视为“仅一级分类”。
4. 一级分类名称全局去重；二级分类只在同一一级分类下去重，因而不同一级分类可以拥有同名二级分类。
5. 删除一级分类时，该一级及其二级中的歌曲都变为“未分类”。删除二级分类时，该二级中的歌曲回到对应一级分类。
6. 分类仍从歌曲记录汇总，不保存空的一级或二级分类；不创建独立 `Category` SwiftData 模型。

## 用户流程

1. 用户点顶部分类菜单，可选择“全部音乐”“未分类”或某个一级分类。
2. 选择一级分类时，列表包含该一级及其全部二级中的歌曲；一级菜单内可继续选择具体二级分类，仅显示该二级歌曲。
3. 用户通过歌曲的“…” → “重命名”打开编辑页，在“分类”区选择或输入一级分类；有一级分类后可选择或输入该一级下的二级分类。
4. 用户可以将歌曲移动到未分类、一级分类或某个二级分类。
5. 在“管理分类”中，用户可重命名或删除一级分类，也可在某一级菜单中重命名或删除它的二级分类。

## 数据契约

```swift
@Model
final class Song {
    var categoryName: String?       // 一级分类；现有字段
    var subcategoryName: String?    // 二级分类；新增可选字段
}
```

- `categoryName == nil` 时，`subcategoryName` 必须为 `nil`。
- `categoryName != nil && subcategoryName == nil` 表示歌曲直接属于一级分类。
- 两个字段都非空时表示歌曲属于具体二级分类。
- 所有名称都去除首尾空白；比较名称时忽略大小写。
- 本版本不改动 `title`、`contentHash`、`storageFileName`、封面、MP3 文件或 MP3 元数据。

## 技术栈与项目结构

- 平台：iOS 17、Swift、SwiftUI、SwiftData；不新增第三方依赖。
- `Models/Song.swift`：两级分类的规范化、查询、移动、改名和删除模型逻辑。
- `App/LibraryView.swift`：层级筛选与分类管理菜单。
- `App/SongEditorView.swift`：一级/二级分类选择与输入。
- `LocalPlayerTests/SongTests.swift`：模型层的分类层级、删除和兼容旧数据测试。
- `docs/decisions/ADR-002-two-level-song-categories.md`：说明为何使用两个字段而不是无限分类树或独立分类模型。

## 代码风格

沿用现有四空格 Swift 风格：模型方法将无效输入保持为不可写入状态，界面保存失败时恢复原数据。

```swift
guard let primaryName = Self.normalizedCategoryName(from: rawPrimaryName) else {
    subcategoryName = nil
    return
}
```

## 边界

- **始终做：** 保持两级上限；兼容现有一级分类；校验分类名称；保存失败时恢复原分类；保持当前筛选范围作为播放队列。
- **先询问：** 第三层或无限层级、独立分类模型、空分类保留、一个歌曲属于多个分类、修改 MP3 标签或增加依赖。
- **绝不做：** 提交音乐、图片、IPA、签名证书、Apple ID 或其他密钥。

## 成功标准

1. 旧版已有的平铺分类在升级后仍显示为一级分类，数据不丢失。
2. 用户可创建、选择、筛选、重命名与删除一级和二级分类。
3. 父级筛选包含其直属歌曲和全部二级歌曲；二级筛选只包含该二级歌曲，播放队列与当前筛选一致。
4. 删除二级后其歌曲回到父级；删除一级后其全部歌曲变为未分类。
5. 重启 App 后两级分类仍保留，且 MP3 文件与元数据未改变。

无；两级上限已确认。
