# LocalPlayer V1.1 规格：分类、重命名与 iPhone 适配

**状态：已批准**
**日期：2026-07-13**

## 目标

让个人本地音乐库可以按自定义分类浏览，并在不改动原 MP3 文件的前提下修改 App 内显示的歌名；同时让 LocalPlayer 在 iPhone 16 Pro 上以完整屏幕尺寸运行。

## 技术边界

- 语言与框架：Swift、SwiftUI、SwiftData；最低 iOS 17。
- 仍只支持本地 MP3；不增加在线音乐、播放列表、账号或云同步。
- `title` 和分类仅是 App 内索引数据：不重命名沙盒中的 MP3、不写入 MP3 标签、更不修改“文件”App 中的原文件。
- 不新增第三方依赖。

## 分类

- 分类为扁平结构，不支持嵌套。
- 每首歌最多属于一个分类；`nil` 表示“未分类”。
- 分类列表由已导入歌曲的分类名称汇总而来，因此不保留空分类。
- 用户可在歌曲操作中输入新分类名称，也可把歌曲移动到已有分类或“未分类”。
- 分类名称会去除首尾空格；相同名称（忽略大小写）不重复创建。
- 改名某个分类时，同类所有歌曲同步更新；删除某个分类时，同类所有歌曲恢复为“未分类”。
- 选择某分类后，列表仅显示该分类歌曲；上一首、下一首和播放结束后的自动切歌只在当前过滤结果中工作。

## 歌曲重命名

- 用户可以在歌曲操作中编辑显示歌名。
- 去除首尾空格后不得为空；保存失败时保留原歌名并显示错误。
- 修改写入 SwiftData，重启 App 后仍保留。
- 重命名不会改变 `storageFileName`、`contentHash`、导入去重规则或 MP3 元数据。

## iPhone 界面与适配

- App 只面向 iPhone，构建产物的 `UIDeviceFamily` 必须为 `[1]`。
- 生成的 `Info.plist` 必须包含 `UILaunchScreen`，避免 iPhone 16 Pro 以兼容尺寸显示根界面。
- 音乐库根界面继续是完整的 `WindowGroup` 内容，不作为 sheet 呈现。
- 所有布局使用 SwiftUI 的安全区、系统 `List` 和动态尺寸；不针对 iPhone 16 Pro 写死像素尺寸。
- 顶部提供分类筛选入口；歌曲行提供清晰的“重命名”“移动到分类”和“删除”操作，并支持 VoiceOver 标签。

## 成功标准

1. 在 iPhone 16 Pro 上，音乐库没有上下黑边或圆角卡片式根窗口，内容铺满安全区内的可用屏幕。
2. 用户可以创建分类、移动歌曲、改名分类、删除分类；删除后相关歌曲显示为“未分类”。
3. 用户可以重命名歌曲；重启 App 后名称与分类仍保留，且原 MP3 文件未改变。
4. 选中分类时，列表和切歌队列只包含该分类歌曲。
5. IPA 内的 `Info.plist` 含 `UIBackgroundModes = [audio]`、`UILaunchScreen` 和仅 iPhone 的设备族；CI 对这些关键值执行检查。

## 命令与验证

本机 Windows 不运行 Xcode；提交后由 GitHub Actions 在 macOS 上执行：

```bash
xcodegen generate
xcodebuild -project LocalPlayer.xcodeproj -scheme LocalPlayer \
  -destination "platform=iOS Simulator,id=<available-simulator-id>" \
  CODE_SIGNING_ALLOWED=NO test
```

真机验证：使用新 IPA 通过 Sideloadly 安装到 iPhone 16 Pro，验证分类、重命名、后台播放与完整屏幕布局。

## 测试策略

- `LocalPlayerTests/SongTests.swift`：验证分类名称与显示歌名的持久化模型行为、输入规范化和过滤/队列逻辑。
- GitHub Actions：生成工程、运行现有单元测试、归档 iPhone App，并检查归档内的 `Info.plist`。
- 真机：验证分类操作、重启后的数据保留，以及回主屏幕/锁屏后仍播放。

## 约束

- **始终**：保留既有 MP3 去重与本地文件安全规则；每个任务通过验证后再进行下一项。
- **先询问**：增加第三方依赖、改变 MP3 原文件或标签、扩展到 iPad。
- **绝不**：提交音乐、IPA、签名证书、Apple ID 或其他密钥。
