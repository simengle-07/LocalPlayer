# LocalPlayer V1 产品规格

## 1. 目标

开发一款仅供个人使用的原生 iPhone 本地 MP3 播放器。

用户可以从“文件”App 选择 MP3 并导入应用，在应用中浏览已导入歌曲、选择歌曲播放；锁屏或切换到其他 App 后，音乐必须继续播放，并能通过锁屏和控制中心控制。

最终产物是一个可由 Sideloadly 安装到个人 iPhone 的 IPA 文件。

## 2. 用户与使用场景

唯一用户：开发者本人。

典型流程：

1. 在 iPhone 的“文件”App 中保存 MP3。
2. 打开 LocalPlayer，点击“导入音乐”。
3. 选择一首或多首 MP3。
4. 在歌曲列表中看到已导入歌曲。
5. 点击歌曲开始播放。
6. 锁屏、切换 App 或使用控制中心继续控制播放。

## 3. V1 功能范围

### 3.1 MP3 导入

- 用户可以通过系统文件选择器选择一个或多个 MP3 文件。
- 导入完成后，应用将音乐保存到自己的本地存储空间。
- 原始 MP3 文件不被修改、移动或删除。
- 重复导入同一首音乐时，应用必须给出明确结果；V1 默认不创建重复歌曲。
- 不支持 MP3 以外的格式。

### 3.2 本地歌曲库

- 显示所有已导入歌曲。
- 每首歌曲至少显示：歌曲名、歌手、时长。
- 优先读取 MP3 内嵌的歌曲名、歌手和封面。
- 元数据缺失时：
  - 歌曲名使用文件名；
  - 歌手显示“未知歌手”；
  - 封面显示默认占位图。
- 默认按歌曲名排序。
- 用户可以删除应用内已导入的歌曲；删除不影响“文件”App 中的原始文件。
- 重启应用后，已导入歌曲仍存在。

### 3.3 播放器

- 点击歌曲即可播放。
- 支持播放、暂停、上一首、下一首。
- 支持拖动进度条跳转播放位置。
- 显示当前歌曲名称、歌手、封面、当前进度和总时长。
- 播放完一首歌曲后，自动播放列表中的下一首。
- 播放到最后一首后停止。

### 3.4 后台与锁屏控制

- 应用进入后台或锁屏后，音乐继续播放。
- 锁屏和控制中心显示当前歌曲信息。
- 锁屏和控制中心支持播放、暂停、上一首、下一首。

## 4. 明确不做的内容

- 在线音乐、音乐搜索、下载、Apple Music 接入。
- 账号、服务器、云同步、社交功能、推荐系统。
- 编辑 MP3 内部标签、重命名原始文件、移动原始文件。
- 播放列表、收藏、歌词、均衡器、CarPlay、无缝播放。
- MP3 之外的音频格式。
- App Store 发布。

## 5. 技术方案

- 开发语言：Swift。
- UI：SwiftUI。
- 音频播放与 MP3 元数据读取：AVFoundation。
- 锁屏和控制中心控制：MediaPlayer。
- 文件导入：SwiftUI 文件选择器。
- 本地数据：应用沙盒中的音乐文件，加上本地歌曲索引。
- 工程生成：XcodeGen，根据 `project.yml` 生成 Xcode 工程。
- 最低系统版本：iOS 17+（暂定）。
- 构建环境：GitHub Actions 的 macOS 运行器。
- 安装方式：构建未签名 IPA，使用 Sideloadly 在个人 iPhone 上签名并安装。

## 6. 项目结构（计划）

```text
App/                    # 应用入口
Features/Library/       # 导入和歌曲列表
Features/Player/        # 播放器界面
Models/                 # Song 等数据模型
Services/Audio/         # 音频播放
Services/Import/        # MP3 导入和元数据读取
Services/MediaControls/ # 锁屏与控制中心控制
Resources/              # 图标、颜色和占位资源
LocalPlayerTests/       # 单元测试
.github/workflows/      # 云端构建 IPA
docs/                   # 规格和设计决策
```

## 7. 测试策略

单元测试：MP3 元数据缺失时的默认显示；
重复导入判断；
歌曲排序；
上一首、下一首和播放结束后的队列行为。

手动真机测试：导入多个 MP3；
删除歌曲后重启应用；
锁屏后持续播放；
锁屏和控制中心控制播放。

云端验证：XcodeGen 能生成工程；
iOS 模拟器构建和单元测试通过；
能产出 IPA 构件。

## 8. 边界与安全规则

音乐文件、IPA、证书、Apple ID 和签名材料绝不提交到 Git。
应用不上传音乐文件或元数据到任何服务器。
导入失败、文件损坏或不支持格式时，必须显示可理解的错误提示。
每完成一个小功能：测试、验证、Git 提交，然后再继续下一项。

## 9. 验收标准

V1 完成时，必须满足：
IPA 能通过 Sideloadly 安装到目标 iPhone。
用户可以从“文件”App 导入多个 MP3。
导入的歌曲在重启 App 后仍显示并可播放。
可以播放、暂停、切歌和拖动进度。
锁屏后音乐不中断，并可在锁屏/控制中心控制播放。
删除歌曲只删除 App 内副本，不影响原始 MP3。

## 10. 待确认事项

App 最终显示名称是否仍为 LocalPlayer。
目标 iPhone 是否为 iOS 17 或更高版本。
重复歌曲的判断依据：完全相同文件，还是歌曲名和歌手相同。
