# LocalPlayer V1.4 实施计划：播放模式

**状态：已实施，等待 GitHub Actions 与真机验证**
**对应规格：** [V1.4 顺序、随机与循环播放规格](v1.4-playback-modes-spec.md)

## 实施顺序

### 任务 22：定义播放模式与可测试的下一首决策

- 新增 `PlaybackMode`：顺序、随机、列表循环、单曲循环。
- 在 `AudioPlayerService` 中集中实现“下一首”决策；明确区分自动结束与手动/远程下一首。
- 用 `UserDefaults` 保存全局模式；测试可注入独立的 defaults suite。
- 为顺序、随机、列表循环、单曲循环及单首随机写单元测试。

### 任务 23：接入播放器、自动连播和远程控制

- 手动下一首、自动连播、锁屏/控制中心下一首共用播放模式决策。
- 上一首保持当前队列顺序，不受随机模式影响。
- 根据模式和队列更新下一首远程命令可用性。

### 任务 24：在正在播放页切换模式

- 在 `NowPlayingView` 增加模式菜单、当前图标和辅助功能文本。
- 切换后即时生效并持久化，不中断当前播放。

## 验证

- Windows 无 Xcode：提交前执行差异、格式和敏感信息检查。
- 推送后由 GitHub Actions 运行 `xcodegen generate` 与 iOS Simulator 单元测试。
- 真机：在 iPhone 16 Pro 用至少三首歌验证四种模式的自动结束、界面下一首、锁屏下一首和重启后保留。

## 边界

- 不修改 MP3 文件、SwiftData schema、CI 或 IPA。
- 不实现随机历史、随机上一首或迷你播放器模式入口。
- 失败时不保留部分播放状态变更。

任务 22–24 已完成实现，等待 GitHub Actions 与 iPhone 16 Pro 真机验收。
