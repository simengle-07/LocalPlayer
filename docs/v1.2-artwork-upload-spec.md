# LocalPlayer V1.2 规格：自定义音乐封面

**状态：已实施，等待 GitHub Actions 验证**
**日期：2026-07-13**

## 目标

让用户为任意已导入歌曲选择、替换或移除 App 内显示的封面。封面会显示在音乐库、迷你播放器和正在播放页，并在重启 App 后保留。

## 已采用的假设

1. 每首歌曲只保存一张静态封面。
2. 封面可以从 iPhone 照片库或“文件”应用选择。
3. 选择的原始图片数据直接存入现有 `Song.artworkData`，不新增模型字段或第三方依赖。
4. 本版本不裁剪、压缩、搜索在线封面，也不修改 MP3 内部标签。

## 技术边界

- 平台：iOS 17、SwiftUI、SwiftData。
- 图片来源：`PhotosPicker` 和 SwiftUI `fileImporter`。
- 文件选择器只接收系统的 `UTType.image`。
- 照片选择器异步读取 `Data`；文件选择器读取安全作用域 URL 的数据。
- 所有外部图片数据必须先由 `UIImage(data:)` 验证；无效数据不覆盖旧封面。

## 用户流程

1. 用户点歌曲行的“…” → “重命名”，进入已有的“编辑歌曲”页。
2. 在“封面”区看到当前封面或默认音乐图标。
3. 用户可选择“从照片选择”或“从文件选择”，选中后立即看到预览。
4. 点“保存”后，歌曲名、分类和封面一起写入 SwiftData。
5. 用户也可点“移除封面”；保存后恢复默认音乐图标。
6. 读取或保存失败时保留原封面，并显示明确错误。

## 数据契约

`Song.artworkData: Data?` 已存在：

- 非空：可由 `UIImage(data:)` 解码的图片数据。
- `nil`：没有自定义封面，界面显示默认音乐图标。
- 本次不改动 `contentHash`、`storageFileName`、MP3 文件或 MP3 元数据。

## 项目结构与代码风格

- `App/SongEditorView.swift`：封面预览、照片/文件选择与保存交互。
- `Models/Song.swift`：提供只改变 App 内封面数据的模型方法。
- `LocalPlayerTests/SongTests.swift`：模型层保存与移除封面的单元测试。
- 使用现有 Swift 命名、四空格缩进、`LocalizedError` 错误信息和 `#expect` 测试风格。

## 命令与验证

Windows 不具备 Xcode；推送后由 GitHub Actions 在 macOS 运行：

```bash
xcodegen generate
xcodebuild -project LocalPlayer.xcodeproj -scheme LocalPlayer \
  -destination "platform=iOS Simulator,id=<available-simulator-id>" \
  CODE_SIGNING_ALLOWED=NO test
```

真机验收：下载新 IPA，通过 Sideloadly 安装到 iPhone 16 Pro，完成“照片选择 → 保存 → 重启 → 文件选择 → 移除封面”一轮测试。

## 边界

- **始终做：** 验证图片数据、保持已有封面直到保存成功、保留现有 MP3 文件保护规则。
- **先询问：** 图像压缩/裁剪、在线封面搜索、修改 MP3 标签、增加第三方库。
- **绝不做：** 提交音乐、图片、IPA、签名证书、Apple ID 或任何密钥。

## 成功标准

1. 用户能从照片库或文件选择有效图片，并在保存后看到封面。
2. 用户能移除封面，并恢复默认图标。
3. 无效图片或读取失败不会删除当前封面。
4. 封面在 App 重启后仍然显示，且 MP3 原文件未被修改。
